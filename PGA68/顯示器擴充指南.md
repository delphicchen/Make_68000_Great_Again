# 68k-nano 加顯示器指南（ESP32 終端機 + 匯流排顯示卡）

68k-nano 本身**沒有顯示功能**，主控台是 16550 UART → J2「5V FTDI SERIAL」。
要接螢幕有兩條路線：

- **路線 A — ESP32/Pico 當「串列終端機」**：不改板、不改 68000 韌體，最快。（建議先做這個）
- **路線 B — 在 68000 匯流排上加記憶體映射顯示卡**：CPU 直接畫面，但工程量大。（先記錄，未來想做圖形/遊戲再用）

---

## 路線 A：ESP32 + FabGL 串列終端機（推薦）

### 原理
ESP32 跑終端機模擬器（FabGL），把 68000 從 UART 送出的字元畫到 VGA 螢幕，
再把 PS/2 鍵盤打的字送回 UART。對 68000 來說它仍然只是在跟一個串列終端機講話，
**板子和韌體完全不用動**。

### 硬體
| 項目 | 建議 |
|------|------|
| 主板 | **TTGO VGA32 / FabGL VGA32**（ESP32 + VGA + 2×PS/2 + audio，為 FabGL 量身做，最省事）|
| 或 | 一般 **classic ESP32**（ESP32-WROOM）自己拉 VGA 電阻 DAC |
| 螢幕 | VGA 顯示器 |
| 鍵盤 | PS/2 鍵盤（或 USB 鍵盤 + PS/2 轉接，需支援 PS/2 模式）|

> ⚠️ FabGL 的 VGA 輸出**只支援初代 ESP32**（用它的 I2S/DAC），
> **不要用 ESP32-S2/S3/C3**。買 VGA32 板最保險。

### 接線（J2 ↔ ESP32）

J2 是 6-pin「5V FTDI SERIAL」，關鍵四隻腳：**TXD（板子送出）、RXD（板子接收）、5V、GND**。
（請以板上絲印為準確認腳序。）

```
68k-nano J2                         ESP32 (3.3V!)
-----------                         -------------
GND  ───────────────────────────── GND
TXD (5V out) ──[ 2kΩ ]──┬───────── RX (UART2 RX)   ← 分壓降到 ~3.3V
                        │
                     [ 1kΩ ]
                        │
                       GND
RXD (5V in)  ─────────────────────  TX (UART2 TX)   ← 3.3V 直接接通常 OK
5V           ─────────────────────  5V / VIN        ← 供電給 ESP32（走板上穩壓）
```

**電平轉換重點（務必）：**
- **68000 TXD 是 5V**，ESP32 腳位只耐 3.3V → 中間要**分壓**（如 2kΩ 上 + 1kΩ 下，
  5V×(1k/(2k+1k))≈… 用 1.8k/3.3k 之類得到 ~3.3V 亦可），或用現成 level shifter。直接接會傷 ESP32。
- ESP32 TXD 3.3V → 16550 RXD：3.3V 對 5V TTL 算高電位，通常可直接接；不穩再加轉換。
- 用 J2 的 5V 餵 ESP32 的 **5V/VIN** 腳（板上有穩壓），別接 3.3V 腳。

### Baud rate
要和 68000 韌體設定的鮑率一致。看 `code/baudrate.py` 與 ROM 韌體設定
（常見 115200 8N1）。FabGL 終端機這端設成同樣值。

### 韌體（Arduino IDE + FabGL）

從 FabGL 內建範例 **`AnsiTerminal`** 改最快。最小骨架（API/腳位以你的 FabGL 版本與板子為準）：

```cpp
#include "fabgl.h"

fabgl::VGAController  DisplayController;
fabgl::PS2Controller  PS2Controller;
fabgl::Terminal       Terminal;

void setup() {
  PS2Controller.begin(PS2Preset::KeyboardPort0, KbdMode::GenerateVirtualKeys);
  DisplayController.begin();
  DisplayController.setResolution(VGA_640x480_60Hz);   // 80x30 字

  Terminal.begin(&DisplayController);
  Terminal.enableCursor(true);

  // 連到接 68000 的 UART2；baud 要對上韌體，rxPin/txPin 依接線
  Terminal.connectSerialPort(115200, true, /*rxPin=*/34, /*txPin=*/2);
  Terminal.write("\e[2J68k-nano terminal ready\r\n");
}

void loop() {
  Terminal.pollSerialPort();
}
```
> VGA32 板的 `AnsiTerminal` 範例已內建正確 VGA/PS2/UART 腳位，直接燒進去多半即用。

### 上電步驟
1. ESP32 燒好終端機韌體，接 VGA 螢幕 + PS/2 鍵盤。
2. J2 ↔ ESP32 按上表接好（含分壓）。
3. 68000 板上電（ROM 已燒韌體）→ 開機訊息應出現在 VGA 螢幕，鍵盤可輸入。

**工作量：約一個週末。68000 板原封不動。**

## 路線 B：68000 匯流排記憶體映射顯示卡（未來參考）

讓 68000 直接寫顯示記憶體 / 顯示控制器暫存器。真正的「顯示卡」，速度快、能做圖形與遊戲，
但難度高很多。

### 三個必須先解決的硬問題

1. **板子沒有拉出完整擴充匯流排**
   對外只有 IDE（J4）、串列（J2）、RTC SPI（J3）。要接顯示卡得從晶片腳飛線拉
   D0–D15、A1–A23、`/AS`、`R/W`、`/UDS`、`/LDS`、CLK、VCC/GND——
   **建議直接改板加一條擴充匯流排排針**，不要在現板上飛一堆線。

2. **`~DTACK` 被硬接到 GND（零等待週期）**
   現設計每個匯流排週期都立即結束（12MHz 下一個週期 ≈ 333ns）。
   新顯示裝置**必須能跟上全速週期**；若用較慢的顯示晶片需要等待狀態，
   就得**切斷 DTACK→GND**，改成由位址解碼 + 裝置 ready 來產生 `~DTACK`。這是最大的改動點。

3. **部分解碼（partial decode）**
   現有：`/ROMSEL=/A23`、`/RAMSEL=A22`、`/UARTSEL=A23·/A22·A21`、`/CARDSEL=A20`。
   記憶體圖中 **`$800000–$8FFFFF` 是 open bus**（A23=1,A22=0,A21=0,A20=0，現有 select 全不命中），
   正好留給擴充。要用它得**自己加一個片選**：
   ```
   /VIDSEL = A23 · /A22 · /A21 · /A20 · /AS   (active low)
   ```
   用一顆 74HC138/GAL/ATF16V8，或 74HC139 的備用半邊產生。

### 匯流排介面要點
- 68000 是 16-bit 資料匯流排，靠 `/UDS`、`/LDS` 做高/低位元組存取——
  8-bit 顯示晶片要像 ROM/RAM 那樣接到 D0–D7 或 D8–D15 並用對應的 DS。
- 顯示裝置未被選到時資料線要 **tri-state**（用 `/VIDSEL` 控 `/CS`/`/OE`）。
- 讀寫方向用 `R/W`（+ `/UDS`/`/LDS`）轉成裝置的 `/RD`、`/WR`。

### 顯示硬體選項
| 方案 | 說明 | 難度 |
|------|------|------|
| **VDP（TMS9918A / V9938 / V9958）** | 自帶 VRAM，匯流排介面簡單（資料線 + mode + CS + RD/WR），輸出 RGB/復合。經典、資料多 | 中（注意存取時序，可能要 DTACK）|
| **並列介面 LCD 控制器**（RA8875 / SSD1963 / ILI9341 並列模式）| 接 TFT/LCD，暫存器映射到匯流排 | 中 |
| **RP2040 / ESP32 當「匯流排週邊」** | 把 MCU 映射到 `$800000`，68000 寫字元/指令，MCU 負責 HDMI/VGA 時序 | 中（不用 FPGA，靈活）|
| **CPLD/FPGA framebuffer + SRAM** | 自製 VGA/HDMI framebuffer，最自由 | 高 |

### 韌體（68000 端）
- 寫驅動：初始化顯示晶片、`putchar`/console 常式；可把韌體主控台從 UART 改導向到顯示。
- 若做 framebuffer，提供畫點/畫字/捲動常式。

### 路線 B 建議
- 想最快又「真顯示卡」感：**VDP（V9958）** 或 **RP2040-當週邊**，
  做成一張小擴充板，加 `/VIDSEL` 解碼，必要時改 DTACK。
- **務必改板拉一條標準擴充匯流排排針**（含 D0–D15、A1–A23、`/AS`、`R/W`、`/UDS`、`/LDS`、
  CLK、`/DTACK`、IRQ、VCC/GND），之後要加任何週邊都方便。

---

## 一句話總結
- **只想看到畫面、能打字** → 路線 A，ESP32/FabGL 當串列終端機，最快、不動板。
- **要 CPU 直接控制圖形/遊戲** → 路線 B，改板加擴充匯流排 + 顯示卡（VDP 或 MCU 當週邊最實際），
  並注意 `~DTACK` 目前接地這件事。
